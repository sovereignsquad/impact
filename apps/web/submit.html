Submit a result (optional)

Submission is off by default. IMPACT never uploads your scan unless you configure an endpoint and explicitly confirm. This page explains what that means and how it relates to future community data.

What submission is

An optional HTTP POST of a privacy-shaped profile to a server you or your organisation configure (IMPACT_SUBMIT_URL). It is designed to support anonymous, aggregate statistics — not to identify you.

What is sent (when you opt in)

  • The same structured profile fields the CLI can write to impact-submission-preview.json — inspect that file before sending.
  • Payload is defined in submission-contract.md.

What is not sent by default

  • Nothing without configuration + consent — default scan is local-only.
  • No raw serials, hardware UUIDs, usernames, hostnames as identifiers, or arbitrary file contents (see privacy docs).

Preview and receipt

The CLI can write impact-submission-preview.json (exact outbound body) and, after an attempt, impact-submission-receipt.json plus a local log under ~/.impact/ — so you keep a record on your machine.

How this powers historical data

Community data on this site will show aggregates only after an ingest pipeline exists, enough submissions exist, and privacy thresholds are met. Your row does not appear instantly on the website; counts update on the server’s schedule and policy.
