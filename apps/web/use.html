Run IMPACT — from curiosity to a clear picture

Four small steps. Everything stays on your disk until you explicitly opt into submission.

  1. Install — Install page (Path B today; npm after #34).
  2. Run one command 
    impact scan --no-submit -o ./reports
  3. Open the HTML report — impact-report.html in your output folder. Same facts as JSON, with provenance and suggested next steps.
  4. (Optional) Inspect impact-profile.json or use the in-browser Profile preview — parsing only, no upload.

Understand the report

  • Host — OS, architecture, coarse memory/disk hints, machine class (not a benchmark).
  • Runtimes — e.g. Ollama installed vs API reachable; MLX may be partial with honest notes.
  • Tools — allowlisted tools found on PATH; others are intentionally ignored.
  • Models — what local APIs returned during the scan; empty does not mean “no models on earth.”

What reachable, partial, and unknown mean

On runtimes, status is operational (e.g. installed but API not responding = installed_unreachable). Partial means we surface honest limits (e.g. MLX pip detection without full model inventory). Unknown means we could not classify reliably on this platform.

Normative vocabulary: schema-semantics-v0.3.md.

What to do next

Follow Suggested next steps in the HTML report (deterministic hints from your profile). Typical actions: start a local runtime, re-scan, or review disk/memory hints — not “buy hardware X.”
