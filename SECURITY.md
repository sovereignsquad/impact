# Security

## Supported versions

Security fixes are applied to the default branch and released as tagged versions once a release cadence exists. See [CHANGELOG.md](CHANGELOG.md).

## Reporting a vulnerability

Please **do not** open a public issue for security-sensitive reports.

Instead, contact the maintainers via GitHub private vulnerability reporting if enabled for this repository, or open a minimal issue asking for a secure contact channel.

We aim to acknowledge reports promptly and coordinate disclosure responsibly.
