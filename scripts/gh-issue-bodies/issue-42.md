## Objective

Preserve the future idea of a drag-and-drop `.dmg` with Applications-folder flow for a later, more user-grade IMPACT distribution path.

## Unified Context

IMPACT is still primarily a local-first CLI product. The near-term product priorities are:
- public npm install truth
- local app usability
- hosted ingest activation
- real aggregate dashboard visibility
- packaging trust and signing discipline

A `.dmg` may become useful later, but it is **not** the right near-term bottleneck to solve. **Signing, notarization, and install truth** come earlier than a polished drag-drop installer.

## Theme / Goal

Keep the future desktop-style macOS install surface visible so it can be revisited when distribution trust and release discipline are mature enough.

## Why this matters

This idea matters because a polished Mac install surface may later improve adoption for less technical users.

## What this does not mean yet

This issue does **not** mean:
- the DMG is the current priority
- the DMG is the product itself
- packaging should outrun signing, notarization, or dashboard activation
- this belongs in current execution

This is explicitly **ideabank**, not current roadmap execution.
