## Objective

Adopt a **Contributor Covenant** (or equivalent) **Code of Conduct** and link it from the **README**, establishing **community guardrails** for a global open source programme.

## Unified Context

IMPACT aims to serve **people everywhere**. A published CoC is baseline **governance** for issues, PRs, and future discussions. This is independent of technical roadmap but **blocks** “community-ready” claims.

## Based On

- CONTRIBUTING.md reference to CoC follow-up
- GitHub community standards

## Problem

- No `CODE_OF_CONDUCT.md`; enforcement path unclear.

## Goal

- Add `CODE_OF_CONDUCT.md` (Contributor Covenant 2.1 or team choice).
- README + CONTRIBUTING link to it.
- Contact / reporting path documented.

## Scope

In scope: file + links. Out of scope: moderation team roster (can be “maintainers via GitHub” initially).

## Execution Prompt

Open PR; reference this issue; comment enforcement email or GitHub reporting flow.

## Constraints

- Must not contradict project **privacy** commitments.

## Acceptance Checks

- [ ] `CODE_OF_CONDUCT.md` present at repo root.
- [ ] README and CONTRIBUTING link to it.
- [ ] Version and attribution of template recorded in file.

## Dependencies

- None technical.

## Risks

- Stale contact info—review yearly.

## Delivery Artifact

- Merged PR + file in repo.
