## Objective

Preserve a future consumer-layer expansion for IMPACT beyond the current CLI and web shell.

## Unified Context

IMPACT is still becoming a strong MLP:
- local scan
- honest install flow
- hosted ingest
- real dashboard data
- release/distribution truth

Broader consumer-product layers may become valuable later, but they are not the present bottleneck. **Accounts, leaderboards, a native GUI, and auto-update** are **out of current execution**; this card is **ideabank**, not the active roadmap queue.

## Theme / Goal

Keep the broader future visible without contaminating current execution focus.

Potential future themes include:
- native desktop shell
- richer user-facing convenience features
- easier release/update channels
- broader public-facing product polish

## Why this matters

These ideas may later improve retention, convenience, and broader adoption once the product foundations are trustworthy.

## What this does not mean yet

This issue does **not** mean:
- native desktop shell now
- broader consumer-layer expansion now
- convenience surfaces before install truth, dashboard truth, and release governance are mature
- this belongs in the active execution queue

This is explicitly **ideabank**, not current roadmap execution.
